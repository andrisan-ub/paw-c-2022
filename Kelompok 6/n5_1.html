Studi kasus registrasi dan login (slide 18-23)


HTML form dapat digunakan untuk mengirim data menggunakan method POST atau method GET. Pada tag form terdapat attribut action dan attribut method, value dari attribut action adalah file php yang akan dikirimkan data, sementara value dari attribut method adalah POST atau GET yang sama sama berfungsi untuk melakukan pengiriman data. Data akan dikirimkan ke file php setelah user menekan tombol submit. Untuk mengakses data di file php kita bisa menggunakan variabel super global yaitu $_GET[“name input”]; untuk method GET dan $_POST[“nameinput”]; untuk method POST.


Pada contoh ini terdapat 2 file yaitu register dan register_process. Register adalah file dimana tag form berada, file ini bisa memiliki ekstensi html ataupun php, namun jika tidak ada kode php didalamnya sebaiknya file ini memiliki ekstensi php. Form pada file register menggunakan method post dan mengirimkan data ke file register_process.php. Data yang dikirimkan adalah username dan password.


Pada file register_process data dari username dan Password diambil menggunakan $_POST[“nameinput”]; dan dipindahkan ke variabel. Data ini digunakan untuk menghasilkan output ucapan selamat datang.


Pada contoh ini terdapat 2 file yaitu login.php dan login_process.php. Tag form berada pada file login.php, data dari file ini dikirimkan menggunakan method POST. File ini memiliki data username dan password yang akan dikirimkan ke file login_process.php.


Pada login_process.php data yang dikirim diambil menggunakan $_POST[“nameinput”]; dan dipindahkan ke varibel. Lalu data dari variabel akan dicek apakah sesuai dengan ketentuan yang ada. Jika sesuai maka akan muncul outpun login succesfull. Jika tidak maka akan muncul output login fails.
