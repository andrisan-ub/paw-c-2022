Introduction

Blade merupakan mesin pembuat template sederhana yang memuat Laravel. Semua template Blade akan dikompilasi ke dalam kode PHP biasa dan di-cache sampai dimodifikasi. Hal tersebut menandakan bahwa Blade tidak membuat aplikasi overhead. Adapun file template Blade dinamai menggunakan “.Blade.php” dan biasanya disimpan dalam direktori Resources/Views. Tampilan Blade dapat dikembalikan dari rute atau pengontrol menggunakan Global View Helper. Data diteruskan ke tampilan Blade menggunakan View Helper’s Second Argument. Contoh: 

Route::get('/', function () {
return view('greeting', ['name' => 'Finn']);
});

Displaying Data

Data yang diteruskan ke tampilan Blade dapat ditampilkan dengan menggunakan kurung kurawal. Contoh: 

Route::get('/', function () {
return view('welcome', ['name' => 'Samantha']);
});

Nama variabel dapat dituliskan seperti berikut: 

Hello, {{ $name }}.

Pernyataan gema Blade {{ }} secara otomatis dikirim melalui fungsi PHP htmlspecialchars untuk mencegah serangan XSS. Pengguna tidak dibatasi untuk menampilkan konten variabel yang diteruskan ke tampilan. Pengguna juga dapat menggemakan hasil fungsi PHP apa pun. Bahkan, pengguna dapat memasukkan kode PHP apa pun yang diinginkan ke dalam pernyataan gema Blade: 

The current UNIX timestamp is {{ time() }}.

HTML Entity Coding

Blade (dan Laravel e helper) akan menggandakan encode entitas HTML secara default. Jika ingin menonaktifkan penyandian ganda, pengguna dapat memanggil metode Blade::withoutDoubleEncoding dari metode boot AppServiceProvider: 

<?php
namespace App/Providers;
use Illuminate\Support\Facades\Blade;
use Illuminate\Support\ServiceProvider;
class ApppServiceProvider extends ServiceProvider
{
/**
* Bootstrap any application services.
*
* @return void
*/
public function boot()
{
Blade::withoutDoubleEncoding();
}
}

Displaying Unescaped Data

Pernyataan Blade {{ }} secara otomatis dikirim melalui fungsi PHP htmlspecialchars untuk mencegah serangan XSS. Jika pengguna tidak ingin datanya di-escape, pengguna dapat menggunakan sintaks berikut: 

Hello, {!! $name !!}.

Blade & JavaScript Frameworks

Karena banyak framework JavaScript juga menggunakan kurung kurawal "{}" untuk menunjukkan ekspresi yang diberikan harus ditampilkan di browser, kita dapat menggunakan simbol “@” untuk memberi tahu compiler Blade bahwa ekspresi harus tetap tidak tersentuh. Sebagai contoh: 

<h1>Laravel<h1>
Hello, @{{ name }}.

Dalam contoh ini, @simbol akan dihapus oleh Blade; namun, {{ name }}ekspresi akan tetap tidak tersentuh oleh compiler Blade, memungkinkannya untuk di render oleh framework JavaScript kita.

Simbol “@” juga dapat digunakan untuk menghindari Blade directive: 

{{-- Blade template --}}
@@if()
{{-- HTML output --}}
@if()

@verbatim Directives

Jika kita menampilkan variabel JavaScript di sebagian besar template kita, kita dapat membungkus HTML di @verbatim Directive sehingga kita tidak harus mengawali setiap pernyataan gema Blade dengan simbol “@”: 

@verbatim
   <div class="container">
      Hello, {{ name }}
   </div>
@endverbatim

Blade Directives

Switch Statements

Pernyataan switch dapat dibangun menggunakan direktif @switch, @case, @break, @default dan @endswitch. Contoh: 

@switch($i)
   @case(1)
       First case...
       @break
   @case(2)
       Second case...
       @break
  @default
      Default case...
@endswitch

Loops

Selain pernyataan bersyarat, Blade juga menyediakan arahan sederhana untuk bekerja dengan struktur loop PHP. Contoh: 

@for ($i = 0; $i < 10; $i++)
   The current value is {{ $i }}
@endfor
@foreach ($users as $user)
   <p>This is user {{ $user->id }}</p>
@endforeach
@forelse ($users as $user)
    <li>{{ $user->name }}</li>
@empty
    <p>No users</p>
@endforelse
@while (true)
    <p>I'm looping forever.</p>
@endwhile

Pengguna dapat menggunakan variabel loop untuk mendapatkan informasi berharga tentang loop saat melakukan iterasi melalui loop @foreach. Saat menggunakan loop, pengguna juga dapat melewati iterasi saat ini atau mengakhiri loop menggunakan arahan @continue dan @break. Contoh: 

@foreach ($users as $user)
   @if ($user->type == 1)
       @continue
   @endif
   <li>{{ $user->name }}</li>
   @if ($user->number == 5)
       @break
  @endif
@endforeach

Pengguna dapat menyertakan kondisi kelanjutan atau pemutusan dalam deklarasi direktif: 

@foreach ($users as $user)
   @continue($user->type == 1)
   <li>{{ $user->name }}</li>
   @break($user->number == 5)
@endforeach

Raw PHP

Dalam beberapa situasi, sangat berguna untuk menyematkan kode PHP ke dalam tampilan kita. Kita dapat menggunakan Blade Directive @php untuk mengeksekusi blok PHP biasa di dalam template kita: 

@php
   $counter = 1;
@endphp

Jika kita hanya perlu menulis satu pernyataan PHP, kita dapat memasukkan pernyataan tersebut dalam @php Directive: 

@php($counter = 1)

Comments

Blade memungkinkan pengguna untuk menuliskan komentar di dalam template. Namun, tidak seperti komentar HTML, komentar Blade tidak disertakan dalam HTML yang dikembalikan oleh aplikasi. Contoh: 

{{-- This comment will not be present in the rendered HTML --}}

Components

Component dan slot memberikan manfaat yang serupa dengan bagian, tata letak, dan termasuk; namun, beberapa mungkin menganggap model component dan slot lebih mudah dipahami. Ada dua pendekatan untuk menulis component: component berbasis kelas dan component anonim.

Untuk membuat komponen berbasis kelas, kita dapat menggunakan perintah artisan “make:component”. Untuk mengilustrasikan cara menggunakan component, kita akan membuat component “Alert” sederhana. Perintah “make:component” akan menempatkan component di direktori “app/View/Components”: 

php artisan make:component Alert

Perintah “make:component” juga akan membuat template tampilan untuk komponen. Tampilan akan ditempatkan di direktori “resources/views/components”. Saat menulis komponen untuk aplikasi kita sendiri, komponen secara otomatis ditemukan di dalam direktori “app/View/Components” dan direktori “resources/views/components”, jadi biasanya tidak diperlukan pendaftaran komponen lebih lanjut.

Kita juga dapat membuat component di subdirektori: 

php artisan make:component Forms/Input

Perintah di atas akan membuat component “Input” di direktori “app/View/Components/Forms” dan tampilan akan ditempatkan di direktori resources/views/components/forms”.

#Mendaftarkan Paket Component Secara Manual

Saat menulis component untuk aplikasi kita sendiri, component secara otomatis ditemukan di dalam direktori “app/View/Components” dan direktori “resources/views/components”.

Namun, jika kita membuat paket yang menggunakan komponen Blade, Anda perlu mendaftarkan kelas component dan alias tag HTML-nya secara manual. Anda biasanya harus mendaftarkan komponen Anda dalam metode “boot” penyedia layanan paket Anda: 

use Illuminate\Support\Facades\Blade;
/**
 * Bootstrap your package's services.
 */
public function boot()
{
    Blade::component('package-alert', Alert::class);
}

Rendering components

Untuk menampilkan suatu komponen, meggunakan salah satu komponen tag dari tamplate blade. Dimulai dengan string x dan disusul dengan kebab case (-) dari komponen kelas seperti contoh: 

<x-alert/>
<x-user-profile/>

Component Attributes

Terkadang kita mungkin perlu menentukan atribut HTML tambahan, seperti “class”, yang bukan merupakan bagian dari data yang diperlukan agar component berfungsi. Biasanya, kita ingin meneruskan atribut tambahan ini ke elemen root template komponen. Misalnya, bayangkan kita ingin me-render component “alert” seperti ini: 

<x-alert type="error" :message="$message" class="mt-4"/>

Semua atribut yang bukan bagian dari konstruktor komponen akan secara otomatis ditambahkan ke "tas atribut" component. Tas atribut ini secara otomatis tersedia untuk komponen melalui variabel “$attributes”. Semua atribut dapat di-render dalam komponen dengan menggunakan variabel ini: 

<div {{ $attributes }}>
   <!-- Component content -->
</div>

Default/Merged Attributes

Terkadang kita mungkin perlu menentukan nilai default untuk atribut atau menggabungkan nilai tambahan ke dalam beberapa atribut component. Untuk mencapai ini, kita dapat menggunakan metode tas atribut “merge”. Metode ini sangat berguna untuk mendefinisikan satu set kelas CSS default yang harus selalu diterapkan ke component: 

<div {{ $attributes->merge(['class' => 'alert alert-'.$type]) }}>
   {{ $message }}
</div>

Jika kita menganggap component ini digunakan seperti ini: 

<x-alert type="error" :message="$message" class="mb-4"/>

HTML akhir yang dirender dari komponen akan muncul seperti berikut: 

<div class="alert alert-error mb-4">
    <!-- Contents of the $message variable -->
</div>

Reserved Keyword

Beberapa kata kunci sudah disediakan secara bawaan sistem untuk merender komponen. Beberapa kata kunci ini tidak bisa dideskripsikan sebagai properti publik atau nama method selain komponen kita yaitu: 

-data
-render
-resolveView
-shouldRender
-view
-withAttributes
-withName

Slots

Pengguna akan sering diminta meneruskan konten tambahan ke komponen melalui "slot". Slot komponen dirender dengan menggemakan variabel $slot. Contoh komponen alert: 

<!-- /resources/views/components/alert.blade.php -->
<div class="alert alert-danger">
   {{ $slot }}
</div>

Konten diteruskan ke slot dengan menyuntikkan konten ke dalam komponen: 

<x-alert>
   <strong>Whoops!</strong> Something went wrong!
</x-alert>

Terkadang komponen mungkin perlu membuat beberapa slot berbeda di lokasi berbeda di dalam komponen. Contoh: 

<!-- /resources/views/components/alert.blade.php -->
<span class="alert-title">{{ $title }}</span>
<div class="alert alert-danger">
   {{ $slot }}
</div>

Pengguna dapat menentukan konten slot yang memiliki nama menggunakan tag x-slot. Konten apa pun yang tidak berada dalam tag x-slot akan diteruskan ke komponen dalam variabel $slot: 

<x-alert>
   <x-slot:title>
       Server Error
   </x-slot>
   <strong>Whoops!</strong> Something went wrong!
</x-alert>

Scoped Slots

Pengguna dapat menggunakan framework JavaScript yang memungkinkan untuk mengakses data atau metode dari komponen yang ada di dalam slot. Pengguna juga dapat mendefinisikan metode atau properti publik pada komponen dan mengakses komponen di dalam slot Anda melalui variabel $component. Dalam contoh ini, komponen x-alert akan diasumsikan memiliki metode formatAlert yang didefinisikan pada kelas komponennya: 

<x-alert>
   <x-slot:title>
       {{ $component->formatAlert('Server Error') }}
   </x-slot>
   <strong>Whoops!</strong> Something went wrong!
</x-alert>

Slot Attributes

Pengguna dapat menetapkan atribut tambahan ke slot seperti nama kelas CSS: 

<x-card class="shadow-sm"></x-card>
   <x-slot:heading class="font-bold"></x-slot:heading>
       Heading
   </x-slot>
   Content
   <x-slot:footer class="text-sm"></x-slot:footer>
       Footer
  </x-slot></code>
</x-card>

Untuk berinteraksi dengan atribut slot, pengguna dapat mengakses properti attributes variabel slot. Contoh kode atribut komponen: 

@props([
     'heading',
     'footer',
])
<div {{ $attributes->class(['border']) }}>
     <h1 {{ $heading->attributes->class(['text-lg']) }}>
         {{ $heading }}
     </h1>
    {{ $slot }}
    <footer {{ $footer->attributes->class(['text-gray-700']) }}>
        {{ $footer }}
    </footer>
</div>

Dynamic Components

Kadang kita juga harus merender komponen tapi tidak tahu komponen mana yang harus dirender hingga runtime. untuk itu, bisa menggunakan dynamic-component untuk merender komponen berdasarkan isi runtime/variabel. 

<x-dynamic-component :component="$componentName" class="mt-4" />

Manually Registering Components

Saat penulisan komponen, komponen dapat ditemukan pada direktori app/View/Components dan resources/views/components. Saat membangun package yang memanfaatkan komponen Blade diperlukan registrasi kelas komponen dan tag HTML secara manual agar Laravel mengetahui letak dari komponen. Untuk melakukan register komponen dilakukan pada method boot pada package service provider. 

use Illuminate\Support\Facades\Blade;
use VendorPackage\View\Components\AlertComponent;
/**
 * Bootstrap your package's services.
 *
 * @return void
 */
public function boot()
{
    Blade::component('package-alert', AlertComponent::class);
}

Saat komponen telah teregistrasi dapat dirender menggunakan tag alias : 

<x-package-alert/>
