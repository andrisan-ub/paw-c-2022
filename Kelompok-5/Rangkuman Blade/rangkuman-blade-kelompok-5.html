Introduction

Blade merupakan mesin pembuat template sederhana yang memuat Laravel. Semua template Blade akan dikompilasi ke dalam kode PHP biasa dan di-cache sampai dimodifikasi. Hal tersebut menandakan bahwa Blade tidak membuat aplikasi overhead. Adapun file template Blade dinamai menggunakan “.Blade.php” dan biasanya disimpan dalam direktori Resources/Views. Tampilan Blade dapat dikembalikan dari rute atau pengontrol menggunakan Global View Helper. Data diteruskan ke tampilan Blade menggunakan View Helper’s Second Argument. Contoh: 

Route::get('/', function () {
return view('greeting', ['name' => 'Finn']);
});

Displaying Data

Data yang diteruskan ke tampilan Blade dapat ditampilkan dengan menggunakan kurung kurawal. Contoh: 

Route::get('/', function () {
return view('welcome', ['name' => 'Samantha']);
});

Nama variabel dapat dituliskan seperti berikut: 

Hello, {{ $name }}.

Pernyataan gema Blade {{ }} secara otomatis dikirim melalui fungsi PHP htmlspecialchars untuk mencegah serangan XSS. Pengguna tidak dibatasi untuk menampilkan konten variabel yang diteruskan ke tampilan. Pengguna juga dapat menggemakan hasil fungsi PHP apa pun. Bahkan, pengguna dapat memasukkan kode PHP apa pun yang diinginkan ke dalam pernyataan gema Blade: 

The current UNIX timestamp is {{ time() }}.

HTML Entity Coding

Blade (dan Laravel e helper) akan menggandakan encode entitas HTML secara default. Jika ingin menonaktifkan penyandian ganda, pengguna dapat memanggil metode Blade::withoutDoubleEncoding dari metode boot AppServiceProvider: 

<?php
namespace App/Providers;
use Illuminate\Support\Facades\Blade;
use Illuminate\Support\ServiceProvider;
class ApppServiceProvider extends ServiceProvider
{
/**
* Bootstrap any application services.
*
* @return void
*/
public function boot()
{
Blade::withoutDoubleEncoding();
}
}

Displaying Unescaped Data

Pernyataan Blade {{ }} secara otomatis dikirim melalui fungsi PHP htmlspecialchars untuk mencegah serangan XSS. Jika pengguna tidak ingin datanya di-escape, pengguna dapat menggunakan sintaks berikut: 

Hello, {!! $name !!}.

Blade & JavaScript Frameworks

Karena banyak framework JavaScript juga menggunakan kurung kurawal "{}" untuk menunjukkan ekspresi yang diberikan harus ditampilkan di browser, kita dapat menggunakan simbol “@” untuk memberi tahu compiler Blade bahwa ekspresi harus tetap tidak tersentuh. Sebagai contoh: 

<h1>Laravel<h1>
Hello, @{{ name }}.

Dalam contoh ini, @simbol akan dihapus oleh Blade; namun, {{ name }}ekspresi akan tetap tidak tersentuh oleh compiler Blade, memungkinkannya untuk di render oleh framework JavaScript kita.

Simbol “@” juga dapat digunakan untuk menghindari Blade directive: 

{{-- Blade template --}}
@@if()
{{-- HTML output --}}
@if()

@verbatim Directives

Jika kita menampilkan variabel JavaScript di sebagian besar template kita, kita dapat membungkus HTML di @verbatim Directive sehingga kita tidak harus mengawali setiap pernyataan gema Blade dengan simbol “@”: 

@verbatim
   <div class="container">
      Hello, {{ name }}
   </div>
@endverbatim

Blade Directives

If Statements

Percabangan ini identik dengan percabangan yang dimiliki PHP yaitu @if, @elseif, @else, @endif contoh implementasinya seperti pada gambar berikut 

@if (count($records) === 1)
   I have one record!
@elseif (count($records) > 1)
   I have multiple records!
@else
   I don't have any records!
@endif

Untuk fleksibilitas blade menyediakan syntax baru di percabangan pada blade yaitu @unless yang merupakan kebalikan dari syntax dari percabangan ifelse implementasinya seperti pada gambar berikut 

@unless (Auth::check())
    You are not signed in.
@endunless

Selain pembahsan yang terdapat diatas, untuk kenyamanan pengguna blade juga menyediakan syntax @isset untuk memberi nilai dan syntax @empty untuk mengosongkan nilai sebuah variabel untuk implementasinya seperti gambar berikut 

@isset($records)
    // $records is defined and is not null...
@endisset

@empty($records)
    // $records is defined and is not null...
@endisset

Switch Statements

Pernyataan switch dapat dibangun menggunakan direktif @switch, @case, @break, @default dan @endswitch. Contoh: 

@switch($i)
   @case(1)
       First case...
       @break
   @case(2)
       Second case...
       @break
  @default
      Default case...
@endswitch

Loops

Selain pernyataan bersyarat, Blade juga menyediakan arahan sederhana untuk bekerja dengan struktur loop PHP. Contoh: 

@for ($i = 0; $i < 10; $i++)
   The current value is {{ $i }}
@endfor
@foreach ($users as $user)
   <p>This is user {{ $user->id }}</p>
@endforeach
@forelse ($users as $user)
    <li>{{ $user->name }}</li>
@empty
    <p>No users</p>
@endforelse
@while (true)
    <p>I'm looping forever.</p>
@endwhile

Pengguna dapat menggunakan variabel loop untuk mendapatkan informasi berharga tentang loop saat melakukan iterasi melalui loop @foreach. Saat menggunakan loop, pengguna juga dapat melewati iterasi saat ini atau mengakhiri loop menggunakan arahan @continue dan @break. Contoh: 

@foreach ($users as $user)
   @if ($user->type == 1)
       @continue
   @endif
   <li>{{ $user->name }}</li>
   @if ($user->number == 5)
       @break
  @endif
@endforeach

Pengguna dapat menyertakan kondisi kelanjutan atau pemutusan dalam deklarasi direktif: 

@foreach ($users as $user)
   @continue($user->type == 1)
   <li>{{ $user->name }}</li>
   @break($user->number == 5)
@endforeach

The Loop Variable

Saat melakukan iterasi melalui sebuah foreach loop, sebuah $loop variabel akan tersedia di dalam loop Anda. Variabel ini menyediakan akses ke beberapa bit informasi yang berguna seperti indeks loop saat ini dan apakah ini adalah iterasi pertama atau terakhir melalui loop 

 @foreach ($users as $user)
 @if ($loop->first)
    This is the first iteration.
 @endif
 @if ($loop->last)
    This is the last iteration.
 @endif
    This is user {{ $user->id }}
 @endforeach

Jika Anda berada dalam loop bersarang, Anda dapat mengakses $loop variabel loop induk melalui parent properti 

 @foreach ($users as $user)
   @foreach ($user->posts as $post)
     @if ($loop->parent->first)
      This is the first iteration of the parent loop. 
   @endif
  @endforeach
 @endforeach
      This is user {{ $user->id }}
 @endforeach

Beberapa variabel loop lainnya yaitu : 

$loop->index     : Indeks dari iterasi loop saat ini (dimulai dari 0).  
$loop->iteration : Iterasi loop saat ini (dimulai dari 1). 
$loop->remaining : Iterasi yang tersisa dalam loop.  
$loop->count     : Jumlah total item dalam array yang diulang.  
$loop->first     : Apakah ini adalah iterasi pertama melalui loop.  
$loop->last      : Apakah ini adalah iterasi terakhir melalui loop.  
$loop->even      : Apakah ini merupakan iterasi genap melalui loop.  
$loop->odd       : Apakah ini iterasi ganjil melalui loop.  
$loop->depth     : Tingkat bersarang dari loop saat ini.  
$loop->parent    : Saat berada dalam loop bersarang, variabel loop induk.

Arahan menerima array kelas di mana kunci array berisi kelas atau kelas yang ingin Anda tambahkan, sedangkan nilainya adalah ekspresi boolean. Jika elemen array memiliki kunci numerik, elemen tersebut akan selalu disertakan dalam daftar kelas yang dirender 

@php
    $isActive = false;
    $hasError = true;
@endphp

<"span @class([

   'text-gray-500' => ! $isActive,
   'bg-red' => $hasError, 
 ])>/span>

<"span class="p-4 text-gray-500 bg-red">

Additional Attributes

Untuk kenyamanan, Anda dapat menggunakan @checkeddirektif untuk dengan mudah menunjukkan jika input kotak centang HTML yang diberikan "dicentang". Arahan ini akan bergema checkedjika kondisi yang diberikan mengevaluasi ke true 

<'input
    type="checkbox"
    name="active"
    value="active"
   @checked(old('active', $user->active)) />

Demikian juga, @selected arahan dapat digunakan untuk menunjukkan apakah opsi pilih yang diberikan harus "dipilih": 

<'select name="version"
   @foreach ($product->versions as $version)
     <'option value="{{ $version }}" @selected(old('version') == $version)>
        {{ $version }}
    <'/option>
   @endforeach
<'/select>

Selain itu, @disabled arahan dapat digunakan untuk menunjukkan apakah elemen yang diberikan harus "dinonaktifkan": 

<' button type="submit" @disabled($errors->isNotEmpty())>Submit<'/button>

Selain itu, @readonly arahan dapat digunakan untuk menunjukkan apakah elemen yang diberikan harus "hanya-baca": 

<'input
    type="email"
    name="email"
    value="email@laravel.com"
    @readonly($user->isNotAdmin()) />

Selain itu, @required arahan dapat digunakan untuk menunjukkan apakah elemen yang diberikan harus "diperlukan" 

<'input
    type="text"
    name="title"
    value="title"
    @required($user->isAdmin()) />

The @once Directive

Arahan @once memungkinkan Anda untuk menentukan bagian dari template yang hanya akan dievaluasi sekali per siklus rendering. Ini mungkin berguna untuk mendorong bagian tertentu dari JavaScript ke dalam header halaman menggunakan stacks . Misalnya, jika Anda merender komponen tertentu dalam satu lingkaran, Anda mungkin hanya ingin mendorong JavaScript ke header saat pertama kali komponen dirender: 

@once
   @push('scripts')
    <'script>
     (custom javascript)
    <'/script>
   @endpush
@endonce

Karena @once direktif sering digunakan bersama dengan @pushor @prepend direktif, @pushOnce dan @prependOnce direktif tersedia untuk kenyamanan Anda: 

@pushOnce('scripts')
    <'script>
     (custom javascript)
    <'/script>
@endPushOnce

Raw PHP

Dalam beberapa situasi, sangat berguna untuk menyematkan kode PHP ke dalam tampilan kita. Kita dapat menggunakan Blade Directive @php untuk mengeksekusi blok PHP biasa di dalam template kita: 

@php
   $counter = 1;
@endphp

Jika kita hanya perlu menulis satu pernyataan PHP, kita dapat memasukkan pernyataan tersebut dalam @php Directive: 

@php($counter = 1)

Comments

Blade memungkinkan pengguna untuk menuliskan komentar di dalam template. Namun, tidak seperti komentar HTML, komentar Blade tidak disertakan dalam HTML yang dikembalikan oleh aplikasi. Contoh: 

{{-- This comment will not be present in the rendered HTML --}}

Components

Component dan slot memberikan manfaat yang serupa dengan bagian, tata letak, dan termasuk; namun, beberapa mungkin menganggap model component dan slot lebih mudah dipahami. Ada dua pendekatan untuk menulis component: component berbasis kelas dan component anonim.

Untuk membuat komponen berbasis kelas, kita dapat menggunakan perintah artisan “make:component”. Untuk mengilustrasikan cara menggunakan component, kita akan membuat component “Alert” sederhana. Perintah “make:component” akan menempatkan component di direktori “app/View/Components”: 

php artisan make:component Alert

Perintah “make:component” juga akan membuat template tampilan untuk komponen. Tampilan akan ditempatkan di direktori “resources/views/components”. Saat menulis komponen untuk aplikasi kita sendiri, komponen secara otomatis ditemukan di dalam direktori “app/View/Components” dan direktori “resources/views/components”, jadi biasanya tidak diperlukan pendaftaran komponen lebih lanjut.

Kita juga dapat membuat component di subdirektori: 

php artisan make:component Forms/Input

Perintah di atas akan membuat component “Input” di direktori “app/View/Components/Forms” dan tampilan akan ditempatkan di direktori resources/views/components/forms”.

#Mendaftarkan Paket Component Secara Manual

Saat menulis component untuk aplikasi kita sendiri, component secara otomatis ditemukan di dalam direktori “app/View/Components” dan direktori “resources/views/components”.

Namun, jika kita membuat paket yang menggunakan komponen Blade, Anda perlu mendaftarkan kelas component dan alias tag HTML-nya secara manual. Anda biasanya harus mendaftarkan komponen Anda dalam metode “boot” penyedia layanan paket Anda: 

use Illuminate\Support\Facades\Blade;
/**
 * Bootstrap your package's services.
 */
public function boot()
{
    Blade::component('package-alert', Alert::class);
}

Rendering components

Untuk menampilkan suatu komponen, meggunakan salah satu komponen tag dari tamplate blade. Dimulai dengan string x dan disusul dengan kebab case (-) dari komponen kelas seperti contoh: 

<x-alert/>
<x-user-profile/>

Passing Data to Components

Anda dapat meneruskan data ke komponen Blade menggunakan atribut HTML. Hard-coded, nilai primitif dapat diteruskan ke komponen menggunakan string atribut HTML sederhana. Ekspresi dan variabel PHP harus diteruskan ke komponen melalui atribut yang menggunakan : karakter sebagai awalan: 

<'x-alert type="error" :message="$message"/>

Anda harus mendefinisikan semua atribut data komponen dalam konstruktor kelasnya. Semua properti publik pada komponen akan secara otomatis tersedia untuk tampilan komponen. Tidak perlu meneruskan data ke tampilan dari metode komponen render:

Short Attribute Syntax

Saat meneruskan atribut ke komponen, Anda juga dapat menggunakan sintaks "atribut pendek". Ini seringkali nyaman karena nama atribut sering cocok dengan nama variabel yang sesuai dengannya: 

{{-- Short attribute syntax... --}}
<'x-profile :$userId :$name />

{{-- Short attribute syntax... --}}
<'x-profile :user-id="$userId" :name="$name" />

Escaping Attribute Rendering

Karena beberapa kerangka kerja JavaScript seperti Alpine.js juga menggunakan atribut awalan titik dua, Anda dapat menggunakan :: awalan titik dua ( ) untuk memberi tahu Blade bahwa atribut tersebut bukan ekspresi PHP. Misalnya, diberikan komponen berikut: 

<'x-button ::class="{ danger: isDeleting }">
   Submit
<'/x-button>

HTML berikut akan dirender oleh Blade: 

<'button :class="{ danger: isDeleting }">
   Submit
<'/button>

Component Methods

Selain variabel publik yang tersedia untuk template komponen Anda, metode publik apa pun pada komponen dapat dipanggil. Misalnya, bayangkan sebuah komponen yang memiliki isSelectedmetode: 

/**
* Determine if the given option is the currently selected option.

*
* @param  string  $option
* @return bool
*/

 public function isSelected($option)
   {
     return $option === $this->selected;
  }

Anda dapat menjalankan metode ini dari templat komponen Anda dengan memanggil variabel yang cocok dengan nama metode: 

<'option {{ $isSelected($value) ? 'selected' : '' }} value="{{ $value }}">
   {{ $label }}
<'/option>

Component Attributes

Terkadang kita mungkin perlu menentukan atribut HTML tambahan, seperti “class”, yang bukan merupakan bagian dari data yang diperlukan agar component berfungsi. Biasanya, kita ingin meneruskan atribut tambahan ini ke elemen root template komponen. Misalnya, bayangkan kita ingin me-render component “alert” seperti ini:

Semua atribut yang bukan bagian dari konstruktor komponen akan secara otomatis ditambahkan ke "tas atribut" component. Tas atribut ini secara otomatis tersedia untuk komponen melalui variabel “$attributes”. Semua atribut dapat di-render dalam komponen dengan menggunakan variabel ini: 

<div {{ $attributes }}>
   <!-- Component content -->
</div>

Default/Merged Attributes

Terkadang kita mungkin perlu menentukan nilai default untuk atribut atau menggabungkan nilai tambahan ke dalam beberapa atribut component. Untuk mencapai ini, kita dapat menggunakan metode tas atribut “merge”. Metode ini sangat berguna untuk mendefinisikan satu set kelas CSS default yang harus selalu diterapkan ke component: 

<div {{ $attributes->merge(['class' => 'alert alert-'.$type]) }}>
   {{ $message }}
</div>

Jika kita menganggap component ini digunakan seperti ini: 

<x-alert type="error" :message="$message" class="mb-4"/>

HTML akhir yang dirender dari komponen akan muncul seperti berikut: 

<div class="alert alert-error mb-4">
    <!-- Contents of the $message variable -->
</div>

Slots

Pengguna akan sering diminta meneruskan konten tambahan ke komponen melalui "slot". Slot komponen dirender dengan menggemakan variabel $slot. Contoh komponen alert: 

<!-- /resources/views/components/alert.blade.php -->
<div class="alert alert-danger">
   {{ $slot }}
</div>

Konten diteruskan ke slot dengan menyuntikkan konten ke dalam komponen: 

<x-alert>
   <strong>Whoops!</strong> Something went wrong!
</x-alert>

Terkadang komponen mungkin perlu membuat beberapa slot berbeda di lokasi berbeda di dalam komponen. Contoh: 

<!-- /resources/views/components/alert.blade.php -->
<span class="alert-title">{{ $title }}</span>
<div class="alert alert-danger">
   {{ $slot }}
</div>

Pengguna dapat menentukan konten slot yang memiliki nama menggunakan tag x-slot. Konten apa pun yang tidak berada dalam tag x-slot akan diteruskan ke komponen dalam variabel $slot: 

<x-alert>
   <x-slot:title>
       Server Error
   </x-slot>
   <strong>Whoops!</strong> Something went wrong!
</x-alert>

Scoped Slots

Pengguna dapat menggunakan framework JavaScript yang memungkinkan untuk mengakses data atau metode dari komponen yang ada di dalam slot. Pengguna juga dapat mendefinisikan metode atau properti publik pada komponen dan mengakses komponen di dalam slot Anda melalui variabel $component. Dalam contoh ini, komponen x-alert akan diasumsikan memiliki metode formatAlert yang didefinisikan pada kelas komponennya: 

<x-alert>
   <x-slot:title>
       {{ $component->formatAlert('Server Error') }}
   </x-slot>
   <strong>Whoops!</strong> Something went wrong!
</x-alert>

Dynamic Components

Kadang kita juga harus merender komponen tapi tidak tahu komponen mana yang harus dirender hingga runtime. untuk itu, bisa menggunakan dynamic-component untuk merender komponen berdasarkan isi runtime/variabel. 

<x-dynamic-component :component="$componentName" class="mt-4" />

Manually Registering Components

Saat penulisan komponen, komponen dapat ditemukan pada direktori app/View/Components dan resources/views/components. Saat membangun package yang memanfaatkan komponen Blade diperlukan registrasi kelas komponen dan tag HTML secara manual agar Laravel mengetahui letak dari komponen. Untuk melakukan register komponen dilakukan pada method boot pada package service provider. 

use Illuminate\Support\Facades\Blade;
use VendorPackage\View\Components\AlertComponent;
/**
 * Bootstrap your package's services.
 *
 * @return void
 */
public function boot()
{
    Blade::component('package-alert', AlertComponent::class);
}

Saat komponen telah teregistrasi dapat dirender menggunakan tag alias : 

<x-package-alert/>

Anonymous Components

Komponen ini menyediakan mekanisme untuk mengatur komponen via satu file. namun, anonymus components memanfaatkan satu view file dan tidak memiliki asosiasi kelas. Cara mendefinisikan anonymous components menempatkan tamplate blade ke dalam resources/views/components. Contoh jika sudah mendefinisikan komponen di resources/views/components/alert.blade.php cukup dirender seperti ini 

<x-alert>

Karakter . juga bisa digunakan jika komponen bersarang . Contoh komponen didefinisikan di resources/views/components/inputs/button.blade.php cukup dirender seperti ini 

<x-inputs.button/>

Anonymous Index Components

Saat terdapat suatu komponen yang dibuat dari banyak blade templates. Kita dapat mengelompokkan template komponen tersebut dalam satu direktori. Contohnya : 

/resources/views/components/accordion.blade.php
/resources/views/components/accordion/item.blade.php

Susunan direktori tersebut dapat dilakukan render komponen accordion dan itemnya seperti : 

<x-accordion>
    <x-accordion.item>
        ...
    </x-accordion.item>
</x-accordion>

Data Properties/Attributes

Kita dapat menentukan artibut yang harus dianggap variabel dengan menggunakan @props di atas komponen tamplate blade. Artibut lain akan tersedia via atribut bag komponen. Jika ingin memberi data variabel sebuah default value, bisa dispesifikkan nama variabel sebagai kunci array dan default value sebagai array value: 

<!-- /resources/views/components/alert.blade.php -->
@props(['type' => 'info', 'message'])
<div {{ $attributes->merge(['class' => 'alert alert-'.$type]) }}>
{{ $message }}
</div>

Kita juga dapat merender seperti ini dari kode diatas: 

<x-alert type="error" :message="$message" class="mb-4"/>

Accessing Parent Data

Terkadang kita perlu mengakses data dari komponen parent di dalam komponen child. Dalam hal ini, kita dapat menggunakan @aware. 

& ltx-menu color="purple"& gt
    & ltx-menu.item& gt...& lt/x-menu.item& gt
    & ltx-menu.item& gt...& lt/x-menu.item& gt
& lt/x-menu& gt

Anonymous Component Namespaces

Dalam pembuatan anonymous Component terletak pada direktori resources/views/components. Akan tetapi, juga dapat melakukan peletakan anonymous components di direktori lain seperti pada resources/views/flights/bookings/components. Hal ini dilakukan dengan menggunakan method anonymousComponentNamespace yang telah disediakan oleh blade. Pada parameter pertama berisi dengan lokasi anonymous componentnya dan parameter kedua berisi namespace nya. 

/**
 * Bootstrap any application services.
 *
 * @return void
 */
public function boot()
{
    Blade::anonymousComponentNamespace('flights.bookings.components', 'flights');
}

Building Layouts

Layouts Using Components

Kebanyakan aplikasi web melakukan maintain layout melalui beberapa halaman. Hal ini tentunya sulit untuk dilakukan maintain pada aplikasi. Dengan blade component dapat mempermudahnya sehingga jadi satu halaman.

(TOLONG INI SUB SUB JUDUL)Membuat Layout komponen

<!-- resources/views/components/layout.blade.php -->
 
<html>
    <head>
        <title>{{ $title ?? 'Todo Manager' }}</title>
    </head>
    <body>
        <h1>Todos</h1>
        <hr/>
        {{ $slot }}
    </body>
</html>

(TOLONG INI SUB SUB JUDUL)Menerapkan Layout komponen

<!-- resources/views/tasks.blade.php -->
 
<x-layout>
    @foreach ($tasks as $task)
        {{ $task }}
    @endforeach
</x-layout>

Layouts Using Template Inheritance

Layout dapat dibuat menggunakan "template inheritance". Cara ini merupakan cara utama dalam pembuatan aplikasi sebelum pengenalan dari komponen. Contoh layout seperti : 

& lt!-- resources/views/layouts/app.blade.php --& gt
 
& lthtml& gt
    & lthead& gt
        & lttitle& gtApp Name - @yield('title')& lt/title& gt
    & lt/head& gt
    & ltbody& gt
        @section('sidebar')
            This is the master sidebar.
        @show
 
        & ltdiv class="container"& gt
            @yield('content')
        & lt/div& gt
    & lt/body& gt
& lt/html& gt

Setelah melakukan pembuatan layout terdapat child page yang mewarisi layout seperti : 

& lt!-- resources/views/child.blade.php --& gt
 
@extends('layouts.app')
 
@section('title', 'Page Title')
 
@section('sidebar')
    @parent
 
    & ltp& gtThis is appended to the master sidebar.& lt/p& gt
@endsection
 
@section('content')
    & ltp& gtThis is my body content.& lt/p& gt
@endsection

Dalam melakukan pewarisan diharuskan menggunakan @extends untuk menunjukkan page mana yang akan diwarisi. Pada contoh tersebut, section sidebar menggunakan @parent untuk menambahkan content pada layout sidebar. @parent akan secara langsung digantikan dengan konten pada layout saat view dirender.

Forms

CSRF Field

Setiap kali pembuatan form diharuskan berisi CSRF token yang tersembunyi. Sehingga CSRF protection middleware dapat melakukan validasi dari request yang ada. Kita dapat menggunakan @crsf untuk menghasilkan token 

<form method="POST" action="/profile">
    @csrf
 
    ...
</form>

Method Field

Sejak form pada HTML tidak dapat membuat request put, patch, dan delete. Kita perlu menambahkan _method tersembunyi. Untuk pembuatan method dapat dilakukan dengan : 

<form action="/foo/bar" method="POST">
    @method('PUT')
 
    ...
</form>

Validation Errors

@error dapat digunakan untuk mengecek apabila terdapat pesan validasi error pada attribute yang ada. Dengan @error kita dapat menampilkan pesan pesan error. 

<!-- /resources/views/post/create.blade.php -->
 
<label for="title">Post Title</label>
 
<input id="title"
    type="text"
    class="@error('title') is-invalid @enderror">
 
@error('title')
    <div class="alert alert-danger">{{ $message }}</div>
@enderror

Sejak @error dicompile pada pernyataan “if”. Kita dapat menggunakan @else untuk merender konten saat tidak terdapat error pada suatu attribute. 

<!-- /resources/views/auth.blade.php -->
 
<label for="email">Email address</label>
 
<input id="email"
    type="email"
    class="@error('email') is-invalid @else is-valid @enderror">

Kita dapat mengirimkan nama dari spesifik error sebagai parameter kedua. 

<!-- /resources/views/auth.blade.php -->
 
<label for="email">Email address</label>
 
<input id="email"
    type="email"
    class="@error('email', 'login') is-invalid @enderror">
 
@error('email', 'login')
    <div class="alert alert-danger">{{ $message }}</div>
@enderror
