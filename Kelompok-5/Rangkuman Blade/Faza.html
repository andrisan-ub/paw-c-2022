Manually Registering Components

Saat penulisan komponen, komponen dapat ditemukan pada direktori app/View/Components dan resources/views/components. Saat membangun package yang memanfaatkan komponen Blade diperlukan registrasi kelas komponen dan tag HTML secara manual agar Laravel mengetahui letak dari komponen. Untuk melakukan register komponen dilakukan pada method boot pada package service provider. 

use Illuminate\Support\Facades\Blade;
use VendorPackage\View\Components\AlertComponent;
/**
 * Bootstrap your package's services.
 *
 * @return void
 */
public function boot()
{
    Blade::component('package-alert', AlertComponent::class);
}

Saat komponen telah teregistrasi dapat dirender menggunakan tag alias : 

<x-package-alert/>

Anonymous Index Components

Saat terdapat suatu komponen yang dibuat dari banyak blade templates. Kita dapat mengelompokkan template komponen tersebut dalam satu direktori. Contohnya : 

/resources/views/components/accordion.blade.php
/resources/views/components/accordion/item.blade.php

Susunan direktori tersebut dapat dilakukan render komponen accordion dan itemnya seperti : 

<x-accordion>
    <x-accordion.item>
        ...
    </x-accordion.item>
</x-accordion>

Accessing Parent Data

Terkadang kita perlu mengakses data dari komponen parent di dalam komponen child. Dalam hal ini, kita dapat menggunakan @aware. 

& ltx-menu color="purple"& gt
    & ltx-menu.item& gt...& lt/x-menu.item& gt
    & ltx-menu.item& gt...& lt/x-menu.item& gt
& lt/x-menu& gt

Anonymous Component Namespaces

Dalam pembuatan anonymous Component terletak pada direktori resources/views/components. Akan tetapi, juga dapat melakukan peletakan anonymous components di direktori lain seperti pada resources/views/flights/bookings/components. Hal ini dilakukan dengan menggunakan method anonymousComponentNamespace yang telah disediakan oleh blade. Pada parameter pertama berisi dengan lokasi anonymous componentnya dan parameter kedua berisi namespace nya. 

/**
 * Bootstrap any application services.
 *
 * @return void
 */
public function boot()
{
    Blade::anonymousComponentNamespace('flights.bookings.components', 'flights');
}

Building Layouts

Layouts Using Components

Kebanyakan aplikasi web melakukan maintain layout melalui beberapa halaman. Hal ini tentunya sulit untuk dilakukan maintain pada aplikasi. Dengan blade component dapat mempermudahnya sehingga jadi satu halaman.

(TOLONG INI SUB SUB JUDUL)Membuat Layout komponen

<!-- resources/views/components/layout.blade.php -->
 
<html>
    <head>
        <title>{{ $title ?? 'Todo Manager' }}</title>
    </head>
    <body>
        <h1>Todos</h1>
        <hr/>
        {{ $slot }}
    </body>
</html>

(TOLONG INI SUB SUB JUDUL)Menerapkan Layout komponen

<!-- resources/views/tasks.blade.php -->
 
<x-layout>
    @foreach ($tasks as $task)
        {{ $task }}
    @endforeach
</x-layout>

Layouts Using Template Inheritance

Layout dapat dibuat menggunakan "template inheritance". Cara ini merupakan cara utama dalam pembuatan aplikasi sebelum pengenalan dari komponen. Contoh layout seperti : 

& lt!-- resources/views/layouts/app.blade.php --& gt
 
& lthtml& gt
    & lthead& gt
        & lttitle& gtApp Name - @yield('title')& lt/title& gt
    & lt/head& gt
    & ltbody& gt
        @section('sidebar')
            This is the master sidebar.
        @show
 
        & ltdiv class="container"& gt
            @yield('content')
        & lt/div& gt
    & lt/body& gt
& lt/html& gt

Setelah melakukan pembuatan layout terdapat child page yang mewarisi layout seperti : 

& lt!-- resources/views/child.blade.php --& gt
 
@extends('layouts.app')
 
@section('title', 'Page Title')
 
@section('sidebar')
    @parent
 
    & ltp& gtThis is appended to the master sidebar.& lt/p& gt
@endsection
 
@section('content')
    & ltp& gtThis is my body content.& lt/p& gt
@endsection

Dalam melakukan pewarisan diharuskan menggunakan @extends untuk menunjukkan page mana yang akan diwarisi. Pada contoh tersebut, section sidebar menggunakan @parent untuk menambahkan content pada layout sidebar. @parent akan secara langsung digantikan dengan konten pada layout saat view dirender.

Forms

CSRF Field

Setiap kali pembuatan form diharuskan berisi CSRF token yang tersembunyi. Sehingga CSRF protection middleware dapat melakukan validasi dari request yang ada. Kita dapat menggunakan @crsf untuk menghasilkan token 

<form method="POST" action="/profile">
    @csrf
 
    ...
</form>

Method Field

Sejak form pada HTML tidak dapat membuat request put, patch, dan delete. Kita perlu menambahkan _method tersembunyi. Untuk pembuatan method dapat dilakukan dengan : 

<form action="/foo/bar" method="POST">
    @method('PUT')
 
    ...
</form>

Validation Errors

@error dapat digunakan untuk mengecek apabila terdapat pesan validasi error pada attribute yang ada. Dengan @error kita dapat menampilkan pesan pesan error. 

<!-- /resources/views/post/create.blade.php -->
 
<label for="title">Post Title</label>
 
<input id="title"
    type="text"
    class="@error('title') is-invalid @enderror">
 
@error('title')
    <div class="alert alert-danger">{{ $message }}</div>
@enderror

Sejak @error dicompile pada pernyataan “if”. Kita dapat menggunakan @else untuk merender konten saat tidak terdapat error pada suatu attribute. 

<!-- /resources/views/auth.blade.php -->
 
<label for="email">Email address</label>
 
<input id="email"
    type="email"
    class="@error('email') is-invalid @else is-valid @enderror">

Kita dapat mengirimkan nama dari spesifik error sebagai parameter kedua. 

<!-- /resources/views/auth.blade.php -->
 
<label for="email">Email address</label>
 
<input id="email"
    type="email"
    class="@error('email', 'login') is-invalid @enderror">
 
@error('email', 'login')
    <div class="alert alert-danger">{{ $message }}</div>
@enderror
