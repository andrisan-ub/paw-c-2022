Components

Component dan slot memberikan manfaat yang serupa dengan bagian, tata letak, dan termasuk; namun, beberapa mungkin menganggap model component dan slot lebih mudah dipahami. Ada dua pendekatan untuk menulis component: component berbasis kelas dan component anonim.

Untuk membuat komponen berbasis kelas, kita dapat menggunakan perintah artisan “make:component”. Untuk mengilustrasikan cara menggunakan component, kita akan membuat component “Alert” sederhana. Perintah “make:component” akan menempatkan component di direktori “app/View/Components”: 

php artisan make:component Alert

Perintah “make:component” juga akan membuat template tampilan untuk komponen. Tampilan akan ditempatkan di direktori “resources/views/components”. Saat menulis komponen untuk aplikasi kita sendiri, komponen secara otomatis ditemukan di dalam direktori “app/View/Components” dan direktori “resources/views/components”, jadi biasanya tidak diperlukan pendaftaran komponen lebih lanjut.

Kita juga dapat membuat component di subdirektori: 

php artisan make:component Forms/Input

Perintah di atas akan membuat component “Input” di direktori “app/View/Components/Forms” dan tampilan akan ditempatkan di direktori resources/views/components/forms”

Mendaftarkan Paket Component Secara Manual

Saat menulis component untuk aplikasi kita sendiri, component secara otomatis ditemukan di dalam direktori “app/View/Components” dan direktori “resources/views/components”.

Namun, jika kita membuat paket yang menggunakan komponen Blade, Anda perlu mendaftarkan kelas component dan alias tag HTML-nya secara manual. Anda biasanya harus mendaftarkan komponen Anda dalam metode “boot” penyedia layanan paket Anda: 

use Illuminate\Support\Facades\Blade;
/**
* Bootstrap any application services.
*/
public function boot()
{
Blade::component('package-alert', Alert::class);
}

Rendering components

Untuk menampilkan suatu komponen, meggunakan salah satu komponen tag dari tamplate blade. Dimulai dengan string x dan disusul dengan kebab case (-) dari komponen kelas seperti contoh: 

<x-alert/>
<x-user-profile/>

Reserved Keyword

Beberapa kata kunci sudah disediakan secara bawaan sistem untuk merender komponen. Beberapa kata kunci ini tidak bisa dideskripsikan sebagai properti publik atau nama method selain komponen kita yaitu: 

-data
-render
-resolveView
-shouldRender
-view
-withAttributes
-withName

Inline Component views

Untuk komponen paling kecil, kita bisa mengembalikan markup komponen dengan metode render : 

/**
* Get the view / contents that represent the component.
*
* @return \Illuminate\View\View|\Closure|string
*/
public function render()
<div class="alert alert-danger">
{{ $slot }}
<div>
blade;
}

Dynamic Components

Kadang kita juga harus merender komponen tapi tidak tahu komponen mana yang harus dirender hingga runtime. untuk itu, bisa menggunakan dynamic-component untuk merender komponen berdasarkan isi runtime/variabel. 

<x-dynamic-component :component="$componentName" class="mt-4" />

Anonymous Components

Komponen ini menyediakan mekanisme untuk mengatur komponen via satu file. namun, anonymus components memanfaatkan satu view file dan tidak memiliki asosiasi kelas. Cara mendefinisikan anonymous components menempatkan tamplate blade ke dalam resources/views/components. Contoh jika sudah mendefinisikan komponen di resources/views/components/alert.blade.php cukup dirender seperti ini 

<x-alert>

Karakter . juga bisa digunakan jika komponen bersarang . Contoh komponen didefinisikan di resources/views/components/inputs/button.blade.php cukup dirender seperti ini 

<x-inputs.button/>

Data Properties/Attributes

Kita dapat menentukan artibut yang harus dianggap variabel dengan menggunakan @props di atas komponen tamplate blade. Artibut lain akan tersedia via atribut bag komponen. Jika ingin memberi data variabel sebuah default value, bisa dispesifikkan nama variabel sebagai kunci array dan default value sebagai array value: 

<!-- /resources/views/components/alert.blade.php -->
@props(['type' => 'info', 'message'])
<div {{ $attributes->merge(['class' => 'alert alert-'.$type]) }}>
{{ $message }}
</div>

Kita juga dapat merender seperti ini dari kode diatas: 

<x-alert type="error" :message="$message" class="mb-4"/>
