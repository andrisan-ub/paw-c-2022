Introduction

Blade merupakan mesin pembuat template sederhana yang memuat Laravel. Semua template Blade akan dikompilasi ke dalam kode PHP biasa dan di-cache sampai dimodifikasi. Hal tersebut menandakan bahwa Blade tidak membuat aplikasi overhead. Adapun file template Blade dinamai menggunakan “.Blade.php” dan biasanya disimpan dalam Resources/Direktori Views. Tampilan Blade dapat dikembalikan dari rute atau pengontrol menggunakan Global View Helper. Data diteruskan ke tampilan Blade menggunakan View Helper’s Second Argument. Contoh: 

Route::get('/', function () {
return view('greeting', ['name' => 'Finn']);
});

Displaying Data

Data yang diteruskan ke tampilan Blade dapat ditampilkan dengan menggunakan kurung kurawal. Contoh: 

Route::get('/', function () {
return view('welcome', ['name' => 'Samantha']);
});

Nama variabel dapat dituliskan seperti berikut: 

Hello, {{ $name }}.

Pernyataan gema Blade {{ }} secara otomatis dikirim melalui fungsi PHP htmlspecialchars untuk mencegah serangan XSS. Pengguna tidak dibatasi untuk menampilkan konten variabel yang diteruskan ke tampilan. Pengguna juga dapat menggemakan hasil fungsi PHP apa pun. Bahkan, pengguna dapat memasukkan kode PHP apa pun yang diinginkan ke dalam pernyataan gema Blade: 

The current UNIX timestamp is {{ time() }}.

HTML Entity Coding

Blade (dan Laravel e helper) akan menggandakan encode entitas HTML secara default. Jika ingin menonaktifkan penyandian ganda, pengguna dapat memanggil metode Blade::withoutDoubleEncoding dari metode boot AppServiceProvider: 

<?php
namespace App/Providers;
use Illuminate\Support\Facades\Blade;
use Illuminate\Support\ServiceProvider;
class ApppServiceProvider extends ServiceProvider
{
/**
* Bootstrap any application services.
*
* @return void
*/
public function boot()
{
Blade::withoutDoubleEncoding();
}
}

Displaying Unescaped Data

Pernyataan Blade {{ }} secara otomatis dikirim melalui fungsi PHP htmlspecialchars untuk mencegah serangan XSS. Jika pengguna tidak ingin datanya di-escape, pengguna dapat menggunakan sintaks berikut: 

Hello, {!! $name !!}.

Blade Directives

Switch Statements

Pernyataan switch dapat dibangun menggunakan direktif @switch, @case, @break, @default dan @endswitch. Contoh: 

@switch($i)
@case(1)
First case...
@break
@case(2)
Second case...
@break
@default
Default case...
@endswitch

Loops

Selain pernyataan bersyarat, Blade juga menyediakan arahan sederhana untuk bekerja dengan struktur loop PHP. Contoh: 

@for ($i = 0; $i < 10; $i++)
The current value is {{ $i }}
@endfor
@foreach ($users as $user)
<p>This is user {{ $user->id }}</p>
@endforeach
@forelse ($users as $user)
<li>{{ $user->name }}</li>
@empty
<p>No users</p>
@endforelse
@while (true)
<p>I'm looping forever.</p>
@endwhile

Pengguna dapat menggunakan variabel loop untuk mendapatkan informasi berharga tentang loop saat melakukan iterasi melalui loop @foreach. Saat menggunakan loop, pengguna juga dapat melewati iterasi saat ini atau mengakhiri loop menggunakan arahan @continue dan @break. Contoh: 

@foreach ($users as $user)
@if ($user->type == 1)
@continue
@endif
<li>{{ $user->name }}</li>
@if ($user->number == 5)
@break
@endif
@endforeach

Pengguna dapat menyertakan kondisi kelanjutan atau pemutusan dalam deklarasi direktif: 

@foreach ($users as $user)
@continue($user->type == 1)
<li>{{ $user->name }}</li>
@break($user->number == 5)
@endforeach

Comments

Blade memungkinkan pengguna untuk menuliskan komentar di dalam template. Namun, tidak seperti komentar HTML, komentar Blade tidak disertakan dalam HTML yang dikembalikan oleh aplikasi. Contoh: 

{{-- This comment will not be present in the rendered HTML --}}

Components

Slots

Pengguna akan sering diminta meneruskan konten tambahan ke komponen melalui "slot". Slot komponen dirender dengan menggemakan variabel $slot. Contoh komponen alert: 

<!-- /resources/views/components/alert.blade.php -->
<div class="alert alert-danger">
{{ $slot }}
</div>

Konten diteruskan ke slot dengan menyuntikkan konten ke dalam komponen: 

<x-alert>
<strong>Whoops!</strong> Something went wrong!
</x-alert>

Terkadang komponen mungkin perlu membuat beberapa slot berbeda di lokasi berbeda di dalam komponen. Contoh: 

<!-- /resources/views/components/alert.blade.php -->
<span class="alert-title">{{ $title }}</span>
<div class="alert alert-danger">
{{ $slot }}
</div>

Pengguna dapat menentukan konten slot yang memiliki nama menggunakan tag x-slot. Konten apa pun yang tidak berada dalam tag x-slot akan diteruskan ke komponen dalam variabel $slot: 

<x-alert>
<x-slot:title>
Server Error
</x-slot>
<strong>Whoops!</strong> Something went wrong!
</x-alert>

Scoped Slots

Pengguna dapat menggunakan framework JavaScript yang memungkinkan untuk mengakses data atau metode dari komponen yang ada di dalam slot. Pengguna juga dapat mendefinisikan metode atau properti publik pada komponen dan mengakses komponen di dalam slot Anda melalui variabel $component. Dalam contoh ini, komponen x-alert akan diasumsikan memiliki metode formatAlert yang didefinisikan pada kelas komponennya: 

<x-alert>
<x-slot:title>
{{ $component->formatAlert('Server Error') }}
</x-slot>
<strong>Whoops!</strong> Something went wrong!
</x-alert>

Slot Attributes

Pengguna dapat menetapkan atribut tambahan ke slot seperti nama kelas CSS: 

<x-card class="shadow-sm"></x-card>
<x-slot:heading class="font-bold"></x-slot:heading>
Heading
</x-slot>
Content
<x-slot:footer class="text-sm"></x-slot:footer>
Footer
</x-slot></code>
</x-card>

Untuk berinteraksi dengan atribut slot, pengguna dapat mengakses properti attributes variabel slot. Contoh kode atribut komponen: 

@props([
'heading',
'footer',
])
<div {{ $attributes->class(['border']) }}>
<h1 {{ $heading->attributes->class(['text-lg']) }}>
{{ $heading }}
</h1>
{{ $slot }}
<footer {{ $footer->attributes->class(['text-gray-700']) }}>
{{ $footer }}
</footer>
</div>
