Introduction

Blade adalah template yang disediakan oleh Laravel untuk memudahkan pemrogram dalam membuat program php. File blade memiliki extension .blade.php dan disimpan dalam directory resources/views. Blade template dapat digunakan bersamaan dengan program php dan framework javascript.

Tanda kurung kurawal {} digunakan untuk menampilkan data di dalam file blade

Dalam program php, untuk menampilkan data digunakan keyword echo. Blade template dapat mempersingkat kode php, karena cukup menggunakan tanda kurung kurawal {}

Blade Directives

Blade template memiliki keyword yang mirip dengan keyword pada php. Setiap keyword pada blade template diawali dengan tanda @

Percabangan

if else

Percabangan if else pada blade template menggunakan keyword @if, @elseif, @else, @endif.

Blade template juga menyediakan keyword @unless. Baris kode di antara keyword @unless dan @endunless akan dijalankan jika kondisi di dalam tanda kurung tidak terpenuhi.

auth

Keyword @auth dan @endauth berfungsi untuk memeriksa apakah pengguna telah terautentifikasi. Sementara itu, keyword @guest dan @endguest digunakan untuk memeriksa apakah pengguna belum terautentifikasi.

enviroment

Kode di antara keyword @env dan @endenv akan dieksekusi jika aplikasi dijalankan dalam lingkungan yang sesuai dengan kondisi di dalam tanda kurung.

section

Keyword @hasSection dan endif digunakan untuk memeriksa apakah suatu section memiliki isi. Sementara itu, keyword @sectionMissing dan @endif digunakan untuk memeriksa apakah section tidak memiliki isi.

switch

Percabangan switch pada blade template sama seperti percabangan switch pada program php. Keyword yang digunakan pada percabangan switch terdiri dari @switch, @case, @break, @default, dan @endswitch

Perulangan

for

Perulangan for menggunakan keyword @for dan endfor.

foreach

Perulangan foreach menggunakan keyword @foreach dan @endforeach.

forelse

Perulangan forelse menggunakan keyword @forelse, @empty, dan @endforelse.

while

Pengulangan while menggunakan keyword @while, dan @endwhile.

Including Subviews

Direktif @include Blade memungkinkan Anda menyertakan tampilan Blade dari dalam tampilan lain. Semua variabel yang tersedia untuk tampilan induk akan tersedia untuk tampilan yang disertakan:

@include('shared.errors')

Meskipun tampilan yang disertakan akan mewarisi semua data yang tersedia di tampilan induk, Anda juga dapat meneruskan larik data tambahan yang harus tersedia untuk tampilan yang disertakan:

@include('view.name', ['status' => 'complete'])

Jika Anda mencoba include tampilan yang tidak ada, Laravel akan membuat kesalahan. Jika Anda ingin menyertakan tampilan yang mungkin ada atau tidak, Anda harus menggunakan direktif @includeIf:

@includeIf('view.name', ['status' => 'complete'])

Jika Anda ingin @include tampilan jika ekspresi boolean yang diberikan bernilai benar atau salah, Anda dapat menggunakan arahan @includeWhen dan @includeUnless:

@includeWhen($boolean, 'view.name', ['status' => 'complete'])

@includeUnless($boolean, 'view.name', ['status' => 'complete'])

Untuk menyertakan tampilan pertama yang ada dari larik tampilan tertentu, Anda dapat menggunakan direktif includeFirst:pression mengevaluasi ke true atau false, Anda dapat menggunakan arahan @includeWhen dan @includeUnless:

@includeFirst(['custom.admin', 'admin'], ['status' => 'complete'])

Rendering Views For Collection

dapat menggabungkan loop dan memasukkan ke dalam satu baris dengan arahan @each Blade:

@each('view.name', $jobs, 'job')

Argumen pertama @each directive adalah tampilan yang akan dirender untuk setiap elemen dalam array atau koleksi. Argumen kedua adalah larik atau koleksi yang ingin Anda ulangi, sedangkan argumen ketiga adalah nama variabel yang akan ditetapkan ke iterasi saat ini dalam tampilan. Jadi, misalnya, jika Anda mengulangi array pekerjaan, biasanya Anda ingin mengakses setiap pekerjaan sebagai variabel pekerjaan dalam tampilan. Kunci array untuk iterasi saat ini akan tersedia sebagai variabel kunci dalam tampilan. Anda juga dapat meneruskan argumen keempat ke @each directive. Argumen ini menentukan tampilan yang akan dirender jika array yang diberikan kosong.

@each('view.name', $jobs, 'job', 'view.empty')

Anonymous Components Namespaces

Komponen anonim biasanya ditentukan dengan menempatkan template Blade di dalam direktori resource/views/components. Namun, sesuai kebutuhan yang mungkin ingin mendaftarkan jalur komponen anonim lainnya dengan Laravel selain jalur default.

Untuk memberi tahu Laravel tentang lokasi komponen anonim ini, Anda dapat menggunakan metode anonymousComponentNamespace yang disediakan oleh fasad Blade. Metode anonymousComponentNamespace menerima "jalur" ke lokasi komponen anonim sebagai argumen pertama dan "namespace" yang komponennya harus ditempatkan sebagai argumen kedua. Seperti pada contoh di bawah, "namespace" akan diawali dengan nama komponen saat komponen dirender.
